## Decisões tomadas

| Decisão | Onde | Racional |
|---------|------|----------|
| **Validator hard vs. soft kinds explícitos** | `validator.HARD_KINDS` / `SOFT_KINDS` constants | Documentação self-evident; PUT handler pode importar e filtrar sem duplicar regra. |
| **`_compile_v2_with_edges` ordena edges + steps** antes de wirear | compiler.py | Determinismo é load-bearing: byte-equivalência v1↔v2 (CA-26) só vale se ordem é estável. |
| **Error handle via state marker + add_conditional_edges** | compiler.py `_wrap_with_error_routing` + `_wire_source_v2` | LangGraph não tem branch nativo para "exception"; marker no state + roteador é o pattern compatível. |
| **Merge `any` semântica V1: pega o primeiro output não-vazio (sort estável)** | compiler.py `_make_merge_any_node` | Cancelamento real exige wrapper externo ao LangGraph (ADR-LOCAL-04 já documentou). V1 entrega semântica utilizável; V2 promove a cancelamento. |
| **Migration preserva `next_step` / `condition.then|else`** | migration_v1_v2.py | Retrocompat 1 release (constitution §11). Drop em sunset (Fase E TASK-E08). |
| **`unauthorized_tool` check só ativa se `allowed_tools` é passado** | validator.py | Sem RBAC service ativo, blocking todo workflow não tem como funcionar. Quando RBAC chegar, router começa a passar a set. |
| **Edges CRUD valida estrutura, não grafo** | edges.py | Separação de concerns: edges.py = data layer; validator.py = graph layer. PUT chama os dois. |
| **Auto-layout cyclic fallback: alphabetic order** | auto_layout.py | Validator deveria ter bloqueado, mas se passar, layout não pode crashar. Best-effort visual. |

## Aprendizados / pegadinhas

- **Compiler retrocompat é frágil sem byte-equivalência testada.** O test `test_compiler_v1_v2_byte_equivalence_linear` introspect `graph.builder.nodes`/`edges` — sem isso, regressão silenciosa em ordem de wire teria passado despercebida. Manter este test verde é o gate canônico para o sunset (Fase E TASK-E08).
- **`add_conditional_edges` precisa que mapping values sejam strings/lists, não funções.** Tentei inicialmente passar callable, falha em runtime. Padrão correto: roteador retorna **string key**, mapping resolve string → target. Documentei no comentário do `err_router`.
- **LangGraph `merge_policy='any'` real exige wrapper externo.** Confirmação prática do que ADR-LOCAL-04 já dizia. Helper `_make_merge_any_node` entrega semântica V1 (pick first non-empty), com TODO claro para promoção. Vale lembrar: V1 ainda não cancela tasks, só descarta resultado.
- **Validator `no_entry_node` co-ocorre com `cycle`.** Quando todos nós formam ciclo, in-degree de cada nó é ≥1, então não há entry. Test `test_validator_no_entry_node_when_every_node_has_inbound` confirma que ambos kinds aparecem — útil para canvas dar dois feedbacks ao usuário.
- **Migration `next_step` para HITL → handle `approved`.** Decisão de constitution §2.4 que casa com semântica histórica: HITL aprovado segue o caminho default. Test `test_migration_condition_steps_emit_then_else` valida o lado condition; deveria ter test análogo para HITL — TODO follow-up.
- **PUT validation rodando após `wf['definition']['steps'] = step_dicts`.** Validator opera sobre o estado *resultante* do PUT. Se rejeitar, mutação acima fica sem reverter — risco. Mitigação aceita: PUT é mostly-additive (steps_count + step config; não toca edges), e canvas auto-save sempre roda /validate antes de PUT. Documentar como TODO se virar problema em produção.
- **Tests com `TestClient` + `_workflows` autouse reset:** funciona porque o conftest do Phase A já zera o store entre tests. Pattern reusable para Phase C tests do frontend que precisam de seed.
